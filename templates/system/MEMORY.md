# MEMORY.md

This file stores durable user-specific memory that should survive across sessions and restarts.

What belongs here:
- stable user preferences
- important names, relationships, and identities the user wants remembered
- long-lived projects, goals, and ongoing situations
- recurring instructions about how to work with the user

What should not be stored here:
- ephemeral chit-chat
- one-off tool outputs
- noisy short-term context that does not matter later

When new important user information appears, update this file carefully using the workspace file tools.
Prefer concise facts over long transcripts.
